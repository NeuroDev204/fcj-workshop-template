This section verifies the product feature built for the user: generating and viewing a 4-week workout plan from the PeriodIQ frontend.

#### Step 1 - Open the Workout Plan page

Log in to PeriodIQ and open **Workout Plan** from the sidebar.

The page contains:

- A summary of the generated plan statistics
- A form for generating a new plan
- A list of existing user plans returned from the API
- A detail entry point for each plan

![Workout Plan generation form](../../../images/5-Workshop/5.4-Nguoi2-RuleEngine/07-product-workout-plan-form.png)

The form collects the minimum input needed by the Rule Engine:

| Input | Purpose |
|---|---|
| `Goal` | Selects the training objective |
| `Fitness Level` | Adjusts volume and intensity |
| `Days/Week` | Controls weekly training frequency |
| `Start Date` | Defines the 4-week date range |
| `Current 1RM` | Calculates target weights |
| `Physical Limitations` | Helps conflict rules reduce risk |
| `Available Equipment` | Filters exercise selection |

#### Step 2 - Generate a 4-week plan

Fill in the form and click:

```text
Generate 4-Week Plan
```

The frontend sends the request to:

```text
POST /api/workoutplans/generate
```

After the API returns successfully, the new plan appears in **My Plans**.

#### Step 3 - Open the generated plan detail

Click **View Detail** on the generated plan.

![Generated 4-week workout plan](../../../images/5-Workshop/5.4-Nguoi2-RuleEngine/08-product-generated-4-week-plan.png)

The detail page shows the final output generated by the Rule Engine:

- 4-week program
- Week-by-week progression
- Day-by-day split
- Exercise list
- Sets and reps
- Target RPE
- Rest time
- Target weight in kilograms

#### Step 4 - Verify periodization result

The generated plan follows the expected periodization structure:

| Week | Expected behavior |
|---|---|
| Week 1 | Build base volume |
| Week 2 | Increase training volume |
| Week 3 | Peak intensity |
| Week 4 | Reduce load for deload |

In the product screenshot, Week 4 is marked as **Deload Week**, proving that the recovery rule was applied instead of increasing load continuously.

#### Step 5 - Verify target weight calculation

Each exercise displays a target weight. For example:

```text
Bench Press: 37.5 kg
Barbell Row: 27.5 kg
Overhead Press: 22.5 kg
```

These numbers are generated from the user's input and the selected plan intensity. This prevents the user from manually guessing working weights.

#### Step 6 - Confirm the user outcome

At the end of this flow, the user can:

- Generate a new 4-week plan
- View saved plans from the API
- Open plan detail
- Follow each training day
- See deload and progression logic clearly

This completes the user-facing part of my role: the Rule Engine is no longer only backend logic; it is visible as a working product feature.
